### T2.2 新增 `src/invoice_ledger/input_profile/embedded_xbrl.py`

职责：检测并解析内嵌 XBRL，输出 InvoiceRecord。接口与规则：

```python
def find_embedded_xbrl(doc) -> tuple[str, bytes] | None      # fitz 文档 → 第一个 .xml 附件
def detect_config_id(xbrl_text: str) -> str | None           # 按命名空间末段匹配（rai→rai_issuer，atr→atr_issuer，einv→einv_*，与 taxonomy 版本解耦）
def parse_xbrl(xbrl_text: str, config: dict) -> dict         # 按 config/xbrl/<id>.xml 的 jsonObject 字段定义提取；xsi:nil 跳过；number→Decimal，date→ISO
def to_invoice_record(parsed: dict, config_id: str, source) -> InvoiceRecord
```

铁路映射（rai_issuer）：

| InvoiceRecord 字段 | 取值 |
| --- | --- |
| invoice_no | ElectronicInvoiceRailwayETicketNumber |
| invoice_date | DateOfIssue |
| buyer_name / buyer_tax_id | NameOfPurchaser / UnifiedSocialCreditCodeOfPurchaser |
| seller_name | NameOfSeller，空则"中国铁路" |
| invoice_type | TypeOfVoucher |
| 明细行 | TypeOfBusiness=售 → 名称"`<DepartureStation>`至`<DestinationStation>`铁路客票"；=退 → "铁路退票费"，备注加"退票费进项抵扣请与主管税务机关确认" |
| line_amount / tax_rate / line_tax_amount / line_total_with_tax | TotalAmountExcludingTax / TaxRate(0.09→"9%") / TaxAmount / Fare |
| 汇总 | amount_total=TotalAmountExcludingTax，tax_total=TaxAmount，total_with_tax=Fare |
| 质量备注 | 注明"数据取自票面内嵌凭证数据" |

- 测试：两张铁路票（乘车 112.00、退票费 38.00）
- 验收：字段与内嵌 XBRL 完全一致（乘车 102.75/9%/9.25/112.00 售；退票 35.85/6%/2.15/38.00 退）；状态 READY；金额勾稽 PASS；重复运行去重跳过

### T2.5 航空 P1a + P6（B 轨修复，最大单点）

**变体路由总原则（适用于所有"新旧并存"票种，航空先行）**：旧票样规则一律保留不删（旧税控票、旧纸行程单仍在报销流通期），但必须变体隔离、排他路由——

1. **数电优先**：命中数电特征词（带括号标题"电子发票（…）"、"民航发展基金"、"增值税税额"、20 位号码）即判数电变体，不再考虑 legacy
2. **legacy 是兜底**：数电特征一个都不命中才落 legacy
3. **特征混杂**（如 OCR 误识导致两变体关键词同时出现）：不硬判，直接进复核（`unmodeled/review`），不写入猜测结果
4. 参照系：`schemas/standard-invoice.yaml` 现有的 `traditional-vat-form`/`digital-invoice-form` 双变体就是正面模式，航空、以及未来任何新旧并存票种都按此模式扩展

- `schemas/air-ticket-itinerary.yaml` 拆变体：`digital-air-itinerary`（命中"电子发票（航空运输电子客票行程单）"或"民航发展基金/增值税税额"）/ `legacy-air-itinerary`（兜底，按上述原则）
- `scheme_extractors/air_ticket.py` 数电分支：
  - invoice_no ←"发票号码："后 20 位（**不取电子客票号码**）；invoice_date ← 优先"填开日期"标签（P6：标签优先，缺失才退启发式）；seller_name ←"填开单位"

## 七、铁律（全程有效）

1. **零新增第三方依赖**：只用现有 venv 已有包 + 标准库。发现"需要装个包"的念头时，回到第四节找替代做法
2. **不过度设计**：不做验签、云查验、OFD 解析、银行/国库凭证、会计入账字段、Java/jar
3. **没有样本不宣称**：无真实样本验证的票种/格式，留接口并在文档注明"未验证"，不写"支持"
4. **双轨一致性**：同一张发票无论走 A 轨还是 B 轨，台账身份（号码、去重键）必须唯一
5. **单一事实源**：只改源仓库；安装副本仅 Phase 3 同步
6. **证据纪律**：每完成一个任务，保存该任务的测试命令与输出摘要（追加到 `实施记录.md`），验收门对比用数据说话
7. **不静默改口径**：凡涉及税务口径的判断（税率、抵扣），只呈现票面/官方数据与来源，抵扣建议一律写"与主管税务机关确认"
